Authentication

The <%= config.projectName %> api is stateless and follows REST principles. The server does therefore not keep any Client Session information. In order to do authentication and authorization each backend request needs to be authenticated.

In production mode the API supports only HTTP bearer-token Authentication for normal requests, in develeopment mode normal requests may also be authticated with HTTP Basic Authentication

The endpoint /login may be used to exchange username/password for a accessToken. So after one call to /login authenticated with username/password (HTTP Basic Auth) a client is able to store an accessToken and can use this accessToken for all further request using HTTP Bearer Token auth.

In production mode it is also possible to use several oAuth providers to gain an accessToken, currently enabled is "Login with Facebook" and "login with your Github Account"
